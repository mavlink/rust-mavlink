fn try_clone(&self) -> Result<Box<dyn SerialPort>>

Cloning is not supported for SerialStream objects

This logic has never really completely worked. Cloned file descriptors in asynchronous code is a semantic minefield. Are you cloning the file descriptor? Are you cloning the event flags on the file descriptor? Both? Itโs a bit of a mess even within one OS, let alone across multiple OSโs

Maybe it can be done with more work, but until a clear use-case is required (or mio/tokio gets an equivalent of the unix AsyncFd for async file handles, see https://github.com/tokio-rs/tokio/issues/3781 and https://github.com/tokio-rs/tokio/pull/3760#issuecomment-839854617) I would rather not have any code available over a kind-of-works-maybe impl. So Iโll leave this code here for now but hard-code it disabled.
